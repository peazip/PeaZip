PeaZip is a free cross-platform data encryption and file archiver utility that provides unified portable GUI for many Open Source technologies like 7-Zip, FreeArc, PAQ, UPX... free alternative to WinRar, WinZip and similar proprietary software.
- Create 7Z, ARC, BZ2, GZ, *PAQ, PEA, QUAD/BALZ, TAR, UPX, WIM, XZ, ZIP files
- Open and extract ACE, ARJ, CAB, DMG, ISO, LHA, RAR, UDF, ZIPX files and more, over 180 archive types supported
Features of PeaZip includes: listing and browsing archives, batch creation and extraction of multiple archives at once, convert files, create self-extracting archives, split and join files, strong encryption with two factor authentication, encrypted password manager, secure deletion, show duplicate files, calculate hash and checksum, export job definition as script.

How to set to ask password for listing, browsing archive content


Under some circumstances, when protecting archive files with encryption, it can be recommended to encrypt the list of files featured inside an archive to hide it from unauthorized users, in order to disallow showing list of archive content before providing password, otherwise, even if the file data itself is encrypted, readable filenames can leak hints to attackers about the nature of the archive.


To prevent this kind of information leak and make archive table of content (TOC) unreadable without knowing the password, disabling listing of the archive when the user will try to open the encrypted archive file, click on the padlock icon to set a password (and optionally a keyfile) for the archive and check "Encrypt also file names" option in the password dialog.
The padlock icon can be found in the status bar, in the file/archive browser, or below the output field in the archive creation interface

 Support for encryption of archive's table of content depends on the archive format design: this feature can't be added to formats lacking it by design, as enforcing it on the archive manager will enable the attacker to bypass this "protection" simply using a different software to list the archive's content.

Support of file names encryption in standards supported by PeaZip:
  • ZIP format lacks this feature, meaning this option will have no effect and archive TOC will always be readable before providing the correct password even if the data is encrypted
  • 7Z and ARC formats support encryption of filenames, so if this option is in use the archive content will be hidden unless correct password is provided for browsing it
  • PEA and ZPAQ format, by design, will never display content of an encrypted archive until correct password is provided, and no option is provided to otherwise allow listing of password-protected files
